1.任务要求及Demo地址

1.1 任务要求

1.头部和底部的黑色区域始终是100%宽
2.页面右侧部分为固定宽度，左侧保持与浏览器窗口变化同步自适应变化
3.左侧的各个模块里面的内容宽度跟随左侧整体宽度同步自适应变化
4.多张图片需要永远都完整展现，所以会随着宽度变窄，从两行变成三行甚至更多，也有可能随着宽度变宽，变成一行

利用position能够实现目标，但是存在着一个问题，父元素不会随着子元素的最高高度变化。也就是说当自适应元素高度大于固定元素的高度时，父元素是可以随着自适应元素的高度变化而变化的，这时候看不出来任何问题。但是，当自适应高度小于固定元素的高度时，父元素高度还是和自适应元素的高度一致，固定元素的部分内容在高度上不会显示出来。也就是说，父元素高度只是随着自适应子元素的高度变化而变化。

3.总结

  1. 类似的页面布局采用float方式最好；
  2. 设置浮动的元素在html代码中必须置于自适应高度的元素之前；
